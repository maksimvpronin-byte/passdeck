# Проверка сборки PassDeck 0.2.0

Проверено в среде Node.js 22:

- `npm run lint` — успешно;
- `npm run typecheck` — успешно;
- `npm test` — 6 тестов успешно;
- `npm run build` — успешно;
- создание, открытие, сохранение и резервные копии KDBX — успешно;
- обычные и защищённые пользовательские поля — успешно;
- редактор пароля с явным показом через отдельный IPC-вызов — lint/typecheck/build успешно;
- Auto-Type использует фиксированную последовательность независимо от старых настроек записи — успешно;
- скрытые значения не попадают в `DatabaseView` и выдаются только отдельным IPC-вызовом;
- сохранение скрытого значения при редактировании и переименовании — успешно;
- Windows runtime Auto-Type и Windows Portable требуют проверки на Windows 10/11.

Ожидаемый артефакт:

```text
release\PassDeck-Portable-0.2.0-x64.exe
```
