# Changelog

## 0.2.15 — in progress

- отсутствующий файл базы теперь отдаёт понятную ошибку `DATABASE_FILE_MISSING` вместо raw `ENOENT`;
- недоступный путь удаляется из recent/restore списков и не всплывает повторно при следующем запуске;
- редактор записи вынесен из `App.tsx` в компонент `EntryEditorModal`;
- confirmation/error-модалки вынесены в переиспользуемые `ConfirmModal` и `ErrorModal`;
- панели групп и списка записей вынесены в `GroupsSidebar` и `EntryList` без переписывания drag-and-drop логики;
- повторяющиеся IPC-обработчики `try/catch -> toApiError` сведены через общий helper без изменения channel names и `ApiResult`;
- renderer больше не отправляет Auto-Type настройки при сохранении записи, дефолтная последовательность остаётся на стороне main/database-service;
- синхронизирована patch-версия проекта.

## 0.2.1

- добавлен macOS Auto-Type по сочетанию `⌘+⌥+A`;
- ввод выполняется в уже активное приложение без его активации или изменения размера;
- перед каждым действием проверяется, что активное приложение не изменилось;
- логин и пароль передаются `osascript` через stdin и не попадают в аргументы процесса;
- добавлена проверка разрешения macOS Accessibility;
- добавлено описание использования Apple Events для системного диалога Automation.

## 0.2.0 — 2026-06-26

- добавлены обычные и защищённые пользовательские поля KDBX;
- защищённые значения передаются интерфейсу только после явного действия;
- существующий пароль редактируется в одном скрытом поле с кнопкой показа;
- добавлены вложения KDBX: множественный выбор, имя и размер, экспорт и удаление;
- установлены лимиты вложений: 25 МБ на файл и 100 МБ на запись;
- запрещено неявное замещение вложения с совпадающим именем;
- добавлено drag-and-drop перемещение записей на группы в дереве;
- перемещение групп, кнопки и диалоги перемещения не добавлялись;
- удаление записей оставлено прямым, с подтверждением и без интерфейса корзины;
- настройки Auto-Type удалены из редактора записи;
- Auto-Type использует фиксированную последовательность `{USERNAME}{TAB}{PASSWORD}{ENTER}`;
- Auto-Type больше не восстанавливает, не активирует и не изменяет размер целевого окна;
- при смене активного окна перед вводом Auto-Type безопасно отменяется;
- стандартизирована Windows-сборка по схеме KubeDeck;
- каноническая логика сборки находится в `scripts/build-portable-windows.ps1`;
- итоговая Windows-сборка создаёт один файл `release/PassDeck-Portable-0.2.0-x64.exe`;
- временные упаковочные каталоги удаляются после успешной сборки;
- обновлены README, roadmap, известные проблемы и сведения о проверке релиза.
