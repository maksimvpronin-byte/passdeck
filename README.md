# PassDeck 0.1.2

PassDeck — локальный portable-менеджер паролей для файлов KeePass `.kdbx`.

Проект использует Electron, React, TypeScript, Vite и Node.js. Отдельного backend-процесса нет. Криптография и операции с файлами выполняются в main-процессе Electron, интерфейс получает данные только через ограниченный IPC API.

## Что уже работает в каркасе

- создание новой базы KDBX 4.1 с AES-256 и Argon2id;
- открытие KDBX 3.1 и 4.x по мастер-паролю;
- несколько баз во вкладках одного окна;
- восстановление вкладок прошлого сеанса в заблокированном виде;
- дерево групп, создание групп;
- просмотр, создание, редактирование и удаление записей;
- логин, пароль, URL, заметки, теги, избранное и срок действия;
- ручное сохранение и сохранение перед закрытием/блокировкой;
- атомарная запись файла;
- две последние зашифрованные резервные копии;
- lock-файл и режим только чтения при занятой базе;
- проверка внешнего изменения файла перед сохранением;
- очистка скопированного пароля и логина из буфера обмена;
- автоблокировка через 10 минут бездействия;
- тёмная и светлая темы;
- настройки рядом с portable-приложением в папке `data`;
- отсутствие телеметрии, сетевой синхронизации и файловых логов;
- unit/integration-тесты KDBX;
- Windows Portable и macOS ARM build-конфигурации;
- Windows Auto-Type через глобальную комбинацию `Ctrl+Alt+A`;
- мгновенный Auto-Type в активное окно без дополнительных подтверждений;
- индивидуальная последовательность Auto-Type для каждой записи.

Полный продуктовый контракт и дальнейший план находятся в [PROJECT_PROMPT.md](PROJECT_PROMPT.md).

Результаты локальной проверки зафиксированы в [BUILD_VALIDATION.md](BUILD_VALIDATION.md).

## Требования

- Windows 10/11 x64 для Windows-сборки;
- macOS 13+ на Apple Silicon для macOS-сборки;
- Node.js 22 LTS;
- npm 10 или новее.

## Windows Portable

В PowerShell из корня проекта:

```powershell
.\build-portable-windows.ps1
```

Результат:

```text
release\PassDeck-Portable-0.1.2-x64.exe
```

Скрипт использует существующий `node_modules`, а при его отсутствии выполняет `npm ci`. Затем запускаются аудит, lint, typecheck, тесты и сборка. Для чистой переустановки используйте `./build-portable-windows.ps1 -CleanInstall`.

## macOS ARM

На MacBook с Apple Silicon:

```bash
chmod +x build-macos.sh
./build-macos.sh
```

Результаты сохраняются в `release/`. Первая сборка не подписана и не проходит notarization, поэтому Gatekeeper может показать предупреждение.

## Windows Auto-Type

1. Откройте и разблокируйте базу.
2. Выберите нужную запись.
3. Перейдите в поле логина сайта или приложения.
4. Нажмите `Ctrl+Alt+A`.
5. PassDeck сразу введёт логин, перейдёт к следующему полю, введёт пароль и нажмёт `Enter`.

Последовательность по умолчанию: `{USERNAME}{TAB}{PASSWORD}{ENTER}`. В редакторе записи её можно изменить, отключить Auto-Type или использовать токены `{URL}` и `{DELAY=500}`. Реализация 0.1.2 предназначена только для Windows. После выбора записи сочетание `Ctrl+Alt+A` сразу вводит данные в активное окно.

## Демонстрационная база

Файл:

```text
test-data/PassDeck-Demo.kdbx
```

Мастер-пароль:

```text
PassDeck-Demo-2026!
```

База содержит только вымышленные данные и не включается в portable-дистрибутив.

## Portable-данные

Рядом с `PassDeck.exe` создаётся структура:

```text
PassDeck-Portable/
├── PassDeck.exe
└── data/
    ├── settings.json
    ├── backups/
    └── recovery/
```

Файловые логи не создаются. Мастер-пароли, ключи шифрования и содержимое записей в `settings.json` не сохраняются.

## Ограничения текущего каркаса

В версии каркаса ещё не завершены импорт/экспорт CSV, вложения в UI, история записей в UI, генератор паролей, импорт/экспорт CSV, аудит слабых и повторяющихся паролей, recovery UI, корзина UI, изменение мастер-пароля и Touch ID. Требования к ним уже зафиксированы в `PROJECT_PROMPT.md` и `docs/ROADMAP.md`.
