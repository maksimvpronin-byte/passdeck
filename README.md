# PassDeck 0.2.7

PassDeck — локальный portable-менеджер паролей для файлов KeePass `.kdbx`.

Приложение построено на Electron, React, TypeScript, Vite и Node.js. Отдельного backend-процесса нет. Криптография, KDBX и файловые операции выполняются в main-процессе Electron, а renderer использует ограниченный типизированный IPC API.

## Возможности версии 0.2.7

- создание новых баз KDBX 4.1 с AES-256 и Argon2id;
- открытие и сохранение KDBX 3.1 и 4.x по мастер-паролю;
- несколько баз во вкладках одного окна;
- восстановление вкладок прошлого сеанса в заблокированном виде;
- дерево групп и создание групп;
- создание, просмотр, редактирование и прямое удаление записей с подтверждением;
- drag-and-drop записей между группами;
- логин, пароль, URL, заметки, теги, избранное и срок действия;
- обычные и защищённые пользовательские поля KDBX;
- вложения KDBX: добавление нескольких файлов, просмотр имени и размера, экспорт и удаление;
- лимиты вложений: до 25 МБ на файл и до 100 МБ на запись;
- ручное сохранение и сохранение перед закрытием или блокировкой;
- атомарная запись базы;
- две последние зашифрованные резервные копии;
- lock-файл и режим только чтения при занятой базе;
- защита от перезаписи базы после внешнего изменения файла;
- очистка скопированного пароля и логина из буфера обмена;
- автоблокировка после бездействия;
- тёмная и светлая темы;
- portable-настройки в папке `data`;
- отсутствие телеметрии, сетевой синхронизации и файловых логов;
- Windows Auto-Type через глобальную комбинацию `Ctrl+Alt+A`;
- macOS Auto-Type через глобальную комбинацию `⌘ Command + ⌥ Option + A`;
- Auto-Type без окна подтверждения и без принудительного восстановления, активации или изменения размера целевого окна;
- разблокировка на macOS через Touch ID после ввода мастер-пароля;
- единый канонический Windows-сборщик и один portable EXE.

История версий записей, корзина, перенос групп и `Save As` намеренно не входят в пользовательский интерфейс PassDeck.

Полный технический контракт находится в [PROJECT_PROMPT.md](PROJECT_PROMPT.md). Результаты проверки сборки зафиксированы в [BUILD_VALIDATION.md](BUILD_VALIDATION.md).

## Требования

- Windows 10/11 x64;
- Node.js 22 LTS;
- npm 10 или новее.

Конфигурация macOS ARM подготовлена для локальной сборки и проверки.

## Windows Portable

Основной запуск из корня проекта:

```powershell
.\build.bat
```

Канонический PowerShell-сборщик:

```powershell
.\scripts\build-portable-windows.ps1
```

Итоговый артефакт:

```text
release\PassDeck-Portable-0.2.7-x64.exe
```

Сборщик использует существующий `node_modules`. Для явной установки зависимостей используйте параметр `-InstallNpmDeps`. Проверки, компиляция и упаковка выполняются по одной цепочке. После упаковки временный каталог `win-unpacked` удаляется, а в `release` остаётся один portable EXE.

## Перемещение записей

Зажмите запись в центральном списке и перетащите её на нужную группу в левой панели. Группа назначения подсветится. Кнопок и диалогов перемещения нет; группы не перетаскиваются.

## Auto-Type

1. Откройте и разблокируйте базу.
2. Выберите запись.
3. Перейдите в поле логина сайта или приложения.
4. Нажмите `Ctrl+Alt+A` на Windows или `⌘ Command + ⌥ Option + A` на macOS.

PassDeck вводит фиксированную последовательность:

```text
{USERNAME}{TAB}{PASSWORD}{ENTER}
```

Перед вводом проверяется, что активное окно не изменилось. PassDeck не вызывает принудительное восстановление или активацию окна и не меняет его размер. Если фокус успел перейти в другое окно, Auto-Type отменяется.

На macOS при первом запуске может потребоваться разрешение. Добавьте PassDeck в:

`Системные настройки → Конфиденциальность и безопасность → Универсальный доступ`

При отдельном запросе «Автоматизация» разрешите PassDeck управлять `System Events`. После изменения разрешений может потребоваться перезапуск PassDeck.

## Вложения

В карточке сохранённой записи можно:

- выбрать один или несколько файлов;
- увидеть имя и размер;
- экспортировать вложение через системный диалог;
- удалить вложение после подтверждения.

Байты вложений не передаются в renderer. Файл с совпадающим именем не заменяет существующее вложение автоматически.

## Демонстрационная база

```text
test-data/PassDeck-Demo.kdbx
```

Мастер-пароль:

```text
PassDeck-Demo-2026!
```

База содержит только вымышленные данные и не включается в portable-дистрибутив.

## Portable-данные

Рядом с portable-приложением используется папка:

```text
data/
├── settings.json
├── backups/
└── recovery/
```

Мастер-пароли, ключи шифрования и содержимое записей в `settings.json` не сохраняются. Файловые логи не создаются.
